Goal

Better the understanding of rhetoric used by politicians through a comprehensive analysis of the tones in their speeches.

What We Did

We used the IBM API to analyze the transcript of speeches to detect emotions based off word choices and clustering. We also used the DeepAffects API to ananalyze inflections in the audio to determine emotions. We then compared the two outputs.

Emotion Key

The IBM API would return analytical, anger, confident, fear, joy, neutral, sadness, tentative and the DeepAffects API would return anger, excited, frustration, happy, neutral, sad. Actually, the IBM API would return no emotions for a sentence it found neutral, but we decided to make that neutral in order to be able to analyze the sentence.

As is, the data is really incomparable, so we decided to transform the data such that uniform emotions are used. We created the keys anger, happy, insecure, neutral, sad, secure. This is how the emotions map over from each kind of API:
